# Documentation

This directory contains all project documentation for oinkerui.

## Structure

- `architecture/` - System architecture, design decisions, and technical specifications
- `api/` - API documentation, endpoint references, and integration guides
- `user/` - End-user documentation, tutorials, and guides

## Documentation Types

### Architecture Documentation
- System design and architecture diagrams
- Database schema and entity relationships
- Component interaction flows
- Technology stack decisions
- Deployment architecture

### API Documentation
- REST API endpoint references
- Request/response schemas
- Authentication and authorization
- Error handling and status codes
- Code examples and integration guides

### User Documentation
- Getting started guide
- Feature tutorials
- Best practices
- Troubleshooting
- FAQ

## Contributing to Documentation

When adding new features or making changes:
1. Update relevant architecture docs if system design changes
2. Update API docs for any endpoint changes
3. Update user docs for any user-facing feature changes
4. Keep documentation in sync with code

## Documentation Tools

- Markdown for all documentation files
- Diagrams using Mermaid or similar tools
- API documentation auto-generated from FastAPI OpenAPI spec
- Version documentation alongside code releases
